Detailed Description

Author
Benjamin Carter

The jackal::Program is an encapsulated class that is used to create and link a variety of GLSLObjects. It is internally used by the Shader classes for compiling and binding the behaviors loaded from the external glsl files.

The Program object also supplies logging of issues and errors that may occur during linkage and compilation. Due to the internal use of the class, it is not exposed to the lua scripting interface.

using namespace jackal;
// Create the program object.
Program program;
program.create(); // Assigns the ID a value.
// Include two external shaders. The VirtualFileSystem can be used.
program.attachShader("~shaders/basic_shader.vertex.glsl", eShaderType::VERTEX);
program.attachShader("~shaders/basic_shader.fragment.glsl", eShaderType::FRAGMENT);
// Compile the program and its shaders.
program.compile();
// Bind the program and render an object.
Program::bind(program);
// render.
Program::unbind();

§ compile()

bool jackal::Program::compile ( )

Compile the Program object.

When the Program is compiled, all of the attached shaders are compiled and linked to the Program ID. When the Program is bound, all of the functionality linked to it will be executed.

Returns
True if the shaders are successfully compiled and linked.

§ create()

void jackal::Program::create ( )

Creates the ID for the Program object.

When the Program object is created, it assigns the ID to a value that can be used to refer to specific behavior on the GPU and attached shaders.

§ destroy()

void jackal::Program::destroy ( )

Destroys the ID of the Program object.

When the destroy method is invoked, if the ID has been assigned a value from the create method, it frees the memory on the GPU.

§ getID()

GLuint jackal::Program::getID ( ) const

Retrieves the unique ID of the Program object.

The ID refers to the unique identifier that is assigned to this Program object. It is assigned when the create method is invoked and refers to the behavior on the GPU side.

Returns
The unique ID of the Program object.

§ recompile()

bool jackal::Program::recompile ( )

Recompiles the Program and its attached shaders.

When the re-compile method is invoked, it will destroy the program and glsl object ID's and re-generate and re-compile them. This method is used so that the glsl files can be hot-swapped and reloaded, so that the application does not have to be restarted.

Unbinds any currently bound programs.
